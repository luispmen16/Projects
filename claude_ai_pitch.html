Chapter 06 — Security & Data Governance

Deployed with
enterprise-grade controls.

Security is not a separate workstream — it is the foundation every deployment wave sits on. Five principles govern how Claude handles your data, and they apply before go-live.

🔒 Five Non-Negotiable Principles

  • No personal data in claude.ai chat without anonymisation first — HR managers may not paste employee names, salaries, or IDs into the interface in plain text
  • Data classification matrix in force before Wave 1: what can go to Claude, in what form, and what never can
  • Claude Code scripts run inside your environment — data stays on your infrastructure; Claude processes the query, not the full raw dataset
  • Agents operate with minimum necessary permissions — the close agent has no access to payroll; the reorder agent has no access to customer pricing
  • Enterprise plan only — Individual Pro accounts do not guarantee data is not used for model training. Non-negotiable before Wave 2

🇪🇺 GDPR — Porto / EU Operations

  • Blue-collar employee data (attendance, clocking, health) requires a lawful basis, data processing record, and DPA with Anthropic under GDPR Article 28
  • Customer names in proposals constitute personal data — anonymise to company-level or ensure privacy notices cover AI-assisted processing
  • Anthropic infrastructure is primarily US-based — Standard Contractual Clauses cover the transfer; your DPO must verify before Wave 2
  • CNPD (Portugal's supervisory authority) is increasingly active — document all AI processing in your Records of Processing Activities (ROPA)
  • Anthropic enterprise DPA confirms conversations are not retained beyond the session — document this to respond to employee GDPR access requests

⚙️ Technical Controls by Wave

  • Wave 1: SSO-linked corporate accounts, data classification policy ratified, DPO sign-off obtained, Anthropic DPA executed before any data-connected use
  • Wave 2: Audit logging on all Claude Code scripts (timestamped, attributed); role-based access controls; ERP connector tested in sandbox before production
  • Wave 3: Per-agent security sign-off document (permissions scope, human-in-loop triggers, rollback procedure); penetration test at Month 6 milestone
  • Wave 4: Full data lineage audit; ESG reporting chain of custody documented for CSRD regulators; annual AI security review added to IT audit calendar

🚨 Top Risks to Manage

  • Data quality failures producing wrong automated outputs — caught too late and surfacing in board packs or regulatory submissions
  • Factory workforce backlash if communication is poor — address proactively in native language, delivered by line managers not HR circulars
  • Over-reliance on agents without human review protocols, creating audit or compliance exposure in financial processes
  • Loss of internal AI owner — the roadmap stalls without a named accountable person; formalise the role before Wave 3
  • The biggest risk is your own people misusing the tool without realising it — one day of scenario-based security training per wave rollout prevents 90% of incidents
